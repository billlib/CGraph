这里之所以 destroy和 delete分开两个循环执行， 是因为当前线程被delete后，还可能存在未被delete的主线程，来steal当前线程的任务 在windows环境下，可能出现问题。 destroy 和 delete 分开之后，不会出现此问题。 感谢 Ryan大佬(\href{https://github.com/ryanhuang}{\texttt{ https\+://github.\+com/ryanhuang}}) 提供的帮助
